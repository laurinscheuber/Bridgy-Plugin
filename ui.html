speed Design Quality Score

The Design Quality Score (DQS) evaluates the maturity and consistency of your design system.

1. Variable Coverage Measures how many properties (colors, spacing, radius) are bound to tokens. Scores decrease as more hard-coded values are found across your nodes.
2. Tailwind Readiness Percentage of variables following Tailwind v4 naming conventions (e.g., --color-primary). Only active when Tailwind v4 export format is selected.
3. Component Hygiene Ratio of component instances vs. raw frames, encouraging composition and modularity.
4. Variable Hygiene Evaluates organization by checking for logical grouping (use of slashes in variable names).
Weights adjust dynamically: all categories share equal weight. When Tailwind v4 is disabled, its weight is redistributed among the remaining categories.

settings Workflow Strategies

ðŸ”„ Merge Request Strategy (Recommended)

Each commit creates a merge request. Multiple commits to the same feature get added to the same MR. Perfect for collaborative review and controlled integration.

âš¡ Direct Commit Strategy

Changes go directly to the main branch. Faster but requires more trust in the design changes. Best for mature teams with established processes.

lightbulb Pro Tips

  • Use descriptive commit messages (e.g., "Update primary colors for better accessibility")
  • Review variables before committing to catch any unintended changes
  • Coordinate with developers for major design system changes
  • Test the setup with small changes before making large updates
  • Keep your Figma variable names consistent and descriptive
